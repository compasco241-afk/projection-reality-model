# Research Questions

**Projection Reality Model — PRM**  
**Questions for development, verification and critical review**

## Purpose

This document collects the main research questions that should guide the further development of the Projection Reality Model.

The purpose is not to present PRM as a completed theory, but to define what must still be clarified, tested, criticized and possibly rejected.

PRM should develop through questions before it develops through stronger claims.

## General Research Direction

PRM currently proposes that reality, information, communication and reasoning can be interpreted through a shared relational pattern based on:

- distinction,
- relation,
- projection,
- resistance,
- change,
- stability,
- Working Relation — WR.

The key question is whether this pattern is genuinely useful, or whether it only redescribes known ideas using new vocabulary.

This must be tested carefully.

## 1. Core Concept Questions

### Distinction

- What exactly counts as a distinction in PRM?
- Is distinction an ontological condition, an epistemic condition, or both?
- Can distinction exist without an observer?
- How does PRM distinguish between physical difference and interpreted difference?
- Can distinction be formalized?

### Relation

- What is a relation in PRM?
- Are relations more fundamental than objects, or only a way of describing objects?
- Can relations exist without stable relata?
- How does PRM handle changing relations?
- Can relations be classified into types?

### Projection

- What exactly is projected?
- What is the difference between projection and appearance?
- Is projection observer-dependent?
- Can projection occur without consciousness?
- Can projection be described mathematically?

### Resistance

- What does resistance mean at different levels of PRM?
- Is resistance always related to stability?
- Can resistance be measured conceptually or formally?
- What is the difference between resistance and inertia?
- Can resistance exist in informational and reasoning structures?

### Change

- What counts as change?
- Is change always relational?
- Can there be change without distinction?
- How does PRM separate actual change from possible change?
- Can change be represented as transformation between WR states?

## 2. Working Relation Questions

WR is currently the most important concept requiring deeper development.

Key questions:

- What exactly is a WR?
- What makes a relational configuration stable enough to be treated as a WR?
- How are the boundaries of a WR defined?
- Can a WR contain other WRs?
- Can multiple WRs form a higher-order WR?
- What is the difference between a WR and a system?
- What is the difference between a WR and an object?
- What is the difference between a WR and a process?
- Can WR be formalized as a graph, network, structure or state-space?
- What would count as a failure of the WR concept?

## 3. Time, Energy and Mass Questions

PRM currently interprets:

- time as ordering of change,
- energy / protoenergy as capacity for transformation,
- mass / protomass as relational resistance.

These interpretations require careful testing.

### Time

- Does PRM add anything useful to the concept of time?
- Is “time as ordering of change” too general?
- How does PRM relate to physical time?
- Can time be understood as a map of possible WR transformations?
- Does PRM distinguish between subjective time, operational time and physical time?

### Energy and protoenergy

- Is protoenergy a useful concept or only a metaphor?
- Can transformative capacity be defined more precisely?
- Can protoenergy be represented without confusing it with physical energy?
- What would make protoenergy measurable or operational?
- Can protoenergy apply to reasoning, information and communication without becoming vague?

### Mass and protomass

- Is protomass a useful way to describe relational resistance?
- How does protomass differ from stability?
- How does protomass differ from inertia?
- Can protomass be used outside physical analogy?
- What would count as excessive or invalid use of the protomass concept?

## 4. Information and Communication Questions

PRM currently interprets information as distinguishable change and communication as propagation of distinguishable change.

Key questions:

- Is “information as distinguishable change” precise enough?
- How does this relate to established information theory?
- What does PRM add beyond known models of signal and information?
- Can information exist without recognition?
- Does recognition require a receiver?
- What is the role of context in information?
- How does PRM explain information loss?
- Can the PRM communication pattern apply across radio, fiber, sound, language and AI without becoming too broad?
- Where does the analogy between technical communication and conceptual communication break down?

## 5. Reasoning and RRG Questions

The Reasoning Reviewer Gate is currently the first practical application of PRM.

Key questions:

- Can reasoning be treated as a WR?
- Can an answer be treated as a projection of a reasoning structure?
- What exactly is a projection error in reasoning?
- How can RRG detect unsupported assumptions?
- Can RRG be converted into a practical checklist?
- Can RRG improve AI answers in real tasks?
- How should RRG handle uncertainty?
- How should RRG handle high-risk domains?
- How can RRG be tested against real examples?
- What would count as failure of RRG?

## 6. Comparison With Existing Frameworks

PRM must be compared with existing ideas to avoid reinventing known concepts.

Important comparison areas:

- systems theory,
- cybernetics,
- information theory,
- structural realism,
- relational ontology,
- process philosophy,
- semiotics,
- graph theory,
- category theory,
- cognitive science,
- AI reasoning and verification,
- philosophy of science.

Key questions:

- What is genuinely new in PRM?
- What is already known under different names?
- Where does PRM overlap with existing frameworks?
- Where does PRM differ meaningfully?
- Could PRM be reduced to systems theory?
- Could PRM be reduced to relational ontology?
- Could PRM be useful as a bridge between several existing frameworks?

## 7. Test Questions

PRM should be tested on concrete cases.

Possible test areas:

- signal transmission,
- internet packet flow,
- information loss,
- AI hallucination,
- legal or formal reasoning,
- physical object stability,
- time-ordering of events,
- transformation of meaning in language,
- system failure and recovery,
- scientific explanation.

For each test case, PRM should ask:

1. What is being distinguished?
2. What relations are involved?
3. What is stable?
4. What changes?
5. What resists change?
6. What is projected?
7. What information is preserved or lost?
8. What does PRM explain better than a normal description?
9. What does PRM fail to explain?

## 8. Counterexample Questions

A serious model must define possible counterexamples.

PRM should ask:

- What would show that PRM is too vague?
- What would show that WR is not useful?
- What would show that projection is only another word for appearance?
- What would show that protoenergy and protomass are misleading?
- What would show that PRM does not add explanatory value?
- What examples cannot be described well by PRM?
- What concepts should be removed if they do not help?

## 9. Formalization Questions

Before PRM can become more rigorous, it needs formalization work.

Key questions:

- Can WR be represented formally?
- Can relations be represented as graph structures?
- Can resistance be modeled as stability under transformation?
- Can projection be modeled as mapping from deeper structure to observable state?
- Can change be modeled as transition between relational states?
- Can information be modeled as preserved distinction?
- Can RRG be modeled as a stability test of reasoning structure?
- What mathematical tools could be appropriate?

Possible tools to investigate:

- graph theory,
- network theory,
- state-space models,
- systems theory,
- category theory,
- logic,
- information theory,
- formal semantics.

## 10. OpenScience Verification Questions

If PRM is later brought into OpenScience or a similar research-review context, the goal should be verification and critique, not promotion.

Questions for external review:

- Is PRM conceptually coherent?
- Are its definitions clear enough?
- Does it overlap with existing theories?
- Does it introduce unnecessary new terminology?
- Which parts are promising?
- Which parts are weak?
- Which parts are too broad?
- What should be formalized first?
- What literature should be compared?
- What would be a reasonable next research step?

## Development Rule

PRM should follow this rule:

> Every strong claim must be preceded by a clear question, a defined concept and a possible way to criticize it.

This protects the project from becoming dogmatic or overextended.

## Current Priority

The current priority is not publication.

The current priority is:

1. clarify WR,
2. test PRM on examples,
3. identify weak points,
4. compare PRM with existing frameworks,
5. develop RRG as a practical review method,
6. prepare PRM for critical review.

## Summary

PRM should be developed through structured questioning.

The most important current research questions are:

- Is WR a useful and precise concept?
- Does PRM add explanatory value?
- Can projection be defined rigorously?
- Can resistance and stability be formalized?
- Can information as distinguishable change be connected to existing theory?
- Can RRG work as a practical reasoning-review tool?
- Can PRM survive counterexamples and critique?

The project should move forward only by testing these questions carefully.
