# Test Cases for PRM

**Projection Reality Model — PRM**  
**Structured test cases for evaluating the usefulness of the model**

## Purpose

This document defines initial test cases for the Projection Reality Model.

The goal is not to prove PRM correct.  
The goal is to test whether PRM provides useful conceptual structure when applied to concrete cases.

Each test case should ask:

1. What is being distinguished?
2. What relations are present?
3. What can be treated as a WR?
4. What changes?
5. What remains stable?
6. What resists change?
7. What is projected?
8. What information is preserved or lost?
9. Does PRM explain anything more clearly than a standard description?
10. Where does PRM become weak, vague or unnecessary?

## Testing Principle

PRM should be tested conservatively.

A test case is useful only if it can show both:

- where PRM helps,
- where PRM fails or adds nothing.

The model should not be protected from criticism.  
A weak test case is valuable if it reveals a weak point in the model.
